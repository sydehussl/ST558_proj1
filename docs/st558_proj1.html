Validating Year

We can validate the user’s inputs one by one, stopping if the function is passed too few options, too many, or invalid ones. We can start by validating the user entered a correct year. They can input any year between 2010 and 2022 inclusive, but not 2020. Adding a check to our function:

query.census <- function(year = 2022, numeric_vars = "AGEP", categorical_vars = "SEX", 
                         geography = "State", specific_geo = "") {
  # check for valid year
  if(year == 2020 | !(year %in% 2010:2022)) stop("Error: Invalid year. Specify a year between 2010 and 2022 (not 2020)")
  
  return("API would be called")
}

Validating Numeric Variables

For numeric variables, we need to consider following:

-The user has to leave the default AGEP or specify their own selection of variables -The additional variables they specify must be valid -PWGTP should always be returned regardless of the user’s selection -There must be at least two numeric variables returned

To satisfy these conditions, we should:

  1. Make sure the user doesn’t pass NULL as their selection
  2. Make sure each variable specified is valid
  3. Add PWGTP to the vector of selected variables

Adding this workflow to our function, we have the following. I’m returning the vector of numeric variables to verify everything is working correctly:

query.census <- function(year = 2022, numeric_vars = "AGEP", categorical_vars = "SEX", 
                         geography = "State", specific_geo = "") {
  # check for valid year
  if(year == 2020 | !(year %in% 2010:2022)) stop("Error: Invalid year. Specify a year between 2010 and 2022 (not 2020)")
  
  # check for valid numeric vars
  numeric_vars_opts <- c("AGEP", "GASP", "GRPIP", "JWAP", "JWDP", "JWMNP")
  if(length(numeric_vars) == 0) stop("No numeric variable selected. Specify at least one.")
  for(i in numeric_vars) {
    if(!(i %in% numeric_vars_opts)) stop("Invalid numeric variable. Specify a single variable or a vector of variables. Options: AGEP, GASP, GRPIP, JWAP, JWDP, JWMNP")
  }
  
  numeric_vars <- c(numeric_vars, "PWGTP")
  
  return(numeric_vars)
}

Validating Geography

We have to consider the following:

  • geography must be State, Region, or District
  • specific_geo must be between 1 and 50 for state, 1 and 4 for region, and 1 and 9 for division
  • specific_geo can be “all” for any selection of geography

Adding these considerations to our function:

query.census <- function(year = 2022, numeric_vars = "AGEP", categorical_vars = "SEX", 
                         geography = "State", specific_geo = 3) {
  # check for valid year
  if(year == 2020 | !(year %in% 2010:2022)) stop("Error: Invalid year. Specify a year between 2010 and 2022 (not 2020)")
  
  # check for valid numeric vars
  numeric_vars_opts <- c("AGEP", "GASP", "GRPIP", "JWAP", "JWDP", "JWMNP")

  }
  
  numeric_vars <- c(numeric_vars, "PWGTP")
  
  # check for valid categorical vars
  categorical_vars_opts <- c("FER", "HHL", "SCH", "SCHL", "SEX")

    if(geography == "Division" & !(specific_geo %in% 1:9)) stop("Invalid division requested. Specify a number 1 to 9 or `all`")
  }
  
  return("API would be called")

Fixing Time Variables

Now we have to deal with the time variables JWAP and JWDP. Looking at the docs, we can see that these variables take on 286 values, so we definitely don’t have the time to hard code their levels.

That being said, I do see a pattern - (almost) every time the variable is incremented, the range is 5 minutes later. For some godforsaken reason, the range jumps 10 minutes when the variable goes from 7 -> 8 and from 10 -> 11. We can work around this quirk, but whoever is responsible deserves a lifetime of perpetually warm pillows. Shame.

Generally, if we set our clock to 12:02am (the middle range of the lowest value), we can add 5 times whatever value of JWAP/JWDP to get the time.

We can test our hypothesis by creating a function and testing

fix.time <- function (time) {
  library(lubridate)
  
  is_zero <- time == 0
  
  scnds_since_midnight <- 120 + (time-1)*300 + ifelse(time > 6, 300, 0) + ifelse(time > 9, 300, 0)
  
  scnds_since_midnight[is_zero] <- 0
  
  return(hms::as_hms(scnds_since_midnight))
}

fix.time <- function (time) {
  library(lubridate)
  
  is_zero <- time == 0
  
  scnds_since_midnight <- 120 + (time-1)*300 + ifelse(time > 6, 300, 0) + ifelse(time > 9, 300, 0) + ifelse(time > 20, 300, 0)
  
  scnds_since_midnight[is_zero] <- 0
  
  return(hms::as_hms(scnds_since_midnight))
}

Calling Multiple Years

Next, we can write a function to get data for multiple years:

query.census.multi <- function(years = 2010:2022, numeric_vars = "AGEP", categorical_vars = "SEX", 
                               geography = "State", specific_geo = 4, return_url = FALSE) {
  if(min(years) < 2010 | max(years) > 2022) {
    stop("Error: Invalid year(s). Specify year(s) between 2010 and 2022")
  }
  
  output <- tibble()
  
  for(i in years[years != 2020]) {
    year_data <- query.census(year = i, numeric_vars = numeric_vars, categorical_vars = categorical_vars,
                              geography = geography, specific_geo = specific_geo)
    
    year_data$year <- i
    
    output <- rbind(output, year_data)
  }
  
  return(output)  
}
